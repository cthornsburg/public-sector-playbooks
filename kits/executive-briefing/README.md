# Executive Briefing Kit

**Problem this solves:** You need to brief leadership quickly without dragging them through technical repo structure.

**Best for:** city/county leaders, boards, executives, supervisors, and technical staff preparing decision-support packets.

**Best front door for non-technical readers:** https://cthornsburg.github.io/public-sector-playbooks/executive/

Use the executive page and pick the relevant problem area:
- incident response

## Good-enough outcome
A non-technical reader should be able to:
- understand the problem
- identify the first document to open
- know which templates or action sheets support decisions
- avoid wandering through the repo blindly
