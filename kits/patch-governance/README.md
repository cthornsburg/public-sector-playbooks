# Patch Governance / KEV-to-Risk Kit

**Problem this solves:** You need a simple way to turn CISA KEV activity into leadership decisions and an IT remediation plan.

**Best for:** security leads, IT managers, public-sector leadership, and anyone trying to shorten patch-risk debates.

**Prefer the web version?** Start here: https://cthornsburg.github.io/public-sector-playbooks/kev-to-risk-pack/

- `slides/kev-to-risk-pack/README.md`
- `slides/kev-to-risk-pack/KEV_TO_RISK_DECK.md`

## Supporting working files

- `templates/kev-to-risk-pack/PATCH_SLA_TABLE.md`

## If you only have 15 minutes
1. Identify internet-facing and high-impact systems.
2. Map current KEV activity to those assets.
3. Use the deck outline to frame urgency.
4. Use the IT action sheet and SLA table to assign remediation.

## Good-enough outcome
You should finish with:
- a leadership-ready patch-risk conversation
- an owner for remediation tracking
- a simple SLA model for action

## Notes
- This kit is optimized for quick executive + IT alignment.
- The hosted page is better for non-technical readers; the repo files are better for maintainers.
