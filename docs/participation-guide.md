# Student Participation Guide

This project welcomes student participation at different skill levels. You do not need to be an expert, and you do not need to submit code to contribute.

## Issues vs pull requests

An **issue** is a note or request. Open an issue when you found something confusing, have an idea, want to ask a question, or want to suggest an improvement.

A **pull request** is a proposed change. Open a pull request when you already edited files and want maintainers to review those edits.

## Examples

- "This sentence is confusing" → open a documentation issue.
- "This lab needs a troubleshooting note" → open a feature/request issue.
- "I fixed the typo" → open a pull request.
- "I do not understand this command" → open a student question issue.
- "I found a better official source" → open an issue or pull request with the citation.

## Contribution ladder

1. **Ask a question** — helps us find unclear material.
2. **Report a small problem** — typo, broken link, outdated screenshot, unclear step.
3. **Suggest an improvement** — new lab idea, resource, checklist, or explanation.
4. **Submit a small edit** — one focused fix is better than a giant first pull request.
5. **Help review** — comment respectfully on issues or PRs when you can verify something.

## What belongs here

This repository focuses on public-sector cybersecurity playbooks, templates, kits, and workbooks.

Good contributions are accurate, safe, practical, and useful to other learners.

## What does not belong here

Do not submit sensitive information, private data, credentials, CTF flags/secrets, or restricted copyrighted material. When in doubt, ask first in an issue without including the sensitive details.

## For instructors

Good classroom-friendly contribution prompts:

- Find and report one unclear instruction.
- Improve one setup or troubleshooting step.
- Add one official reference link and explain why it helps.
- Convert a confusing paragraph into a checklist.
- Review a peer's pull request for clarity and safety.

## Before you submit

- Did you remove sensitive data?
- Did you keep the change focused?
- Did you explain why the change helps?
- Did you cite sources when needed?
